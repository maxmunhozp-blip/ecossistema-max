# Alex Hormozi — $100M Offers for Sales Pages

> Reference file for sales page copy generation. Claude should apply these frameworks
> when building offers, pricing sections, guarantees, and bonus stacks.

---

## 1. Value Equation

The core formula that determines how valuable an offer feels:

```
            Dream Outcome × Perceived Likelihood of Achievement
Value = ─────────────────────────────────────────────────────────────
              Time Delay × Effort & Sacrifice
```

### How to maximize each variable:

| Variable | Increase Value By... | Page Section |
|----------|---------------------|--------------|
| **Dream Outcome** (↑) | Pintar o resultado ideal com detalhes vívidos e específicos | Hero, Transformação |
| **Perceived Likelihood** (↑) | Provas sociais, depoimentos, garantia, credenciais | Provas, Depoimentos |
| **Time Delay** (↓) | Mostrar velocidade do resultado, quick wins, acesso imediato | Método, Oferta |
| **Effort & Sacrifice** (↓) | Done-for-you, templates, atalhos, suporte incluso | Bônus, Oferta |

### Application rule:
> Every section of the sales page should move at least ONE variable in the right direction. If a section doesn't increase the numerator or decrease the denominator, cut it.

## 2. Grand Slam Offer Checklist

A Grand Slam Offer is so good the prospect feels stupid saying no.

### Checklist for sales page copy:

- [ ] **Resultado claro e específico** — "Você vai [resultado] em [prazo]"
- [ ] **Mecanismo único** — Por que SEU método funciona diferente
- [ ] **Prova irrefutável** — Resultados reais, números, screenshots
- [ ] **Garantia que remove risco** — Mais forte que "dinheiro de volta"
- [ ] **Bônus que resolvem objeções** — Cada bônus mata uma desculpa
- [ ] **Urgência real** — Motivo legítimo pra agir AGORA
- [ ] **Preço ancorado** — Valor percebido 10x maior que o preço
- [ ] **Entrega imediata** — Acesso instantâneo ou rápido

### The "Stupid Test":
> Leia a oferta completa e pergunte: "Alguém se sentiria BURRO recusando isso?" Se não, a oferta precisa ser mais forte.

## 3. Guarantee Types

### 3.1 Unconditional (Incondicional)
```
Garantia de [X] dias sem perguntas.
Se por QUALQUER motivo você não gostar, devolvemos 100% do seu investimento.
Sem burocracia. Sem perguntas. Sem ressentimentos.
```

### 3.2 Conditional (Condicional)
```
Se você aplicar [ação específica] por [prazo] e não conseguir [resultado],
devolvemos 100% do seu investimento + R$ [valor extra] pelo tempo perdido.
```

### 3.3 Anti-Guarantee (Anti-Garantia)
```
NÃO oferecemos garantia.
Porque queremos apenas pessoas comprometidas com resultado.
Se você precisa de uma "porta dos fundos", esse programa não é pra você.
```
> Use only for high-ticket offers with strong brand authority.

### 3.4 Performance-Based (Baseada em Performance)
```
Se você não [resultado mensurável] em [prazo], continuamos trabalhando com você
GRATUITAMENTE até você conseguir. Sem custo adicional. Sem limite de tempo.
```

### Guarantee Naming Framework:
Give the guarantee a memorable name that reinforces the promise:

| Pattern | Example |
|---------|---------|
| "Garantia [Resultado]" | "Garantia Resultado em 90 Dias" |
| "Garantia [Sentimento]" | "Garantia Tranquilidade Total" |
| "Garantia [Número]x" | "Garantia Tripla de Satisfação" |
| "Garantia [Metáfora]" | "Garantia Cofre Blindado" |

## 4. Bonus Stacking Strategy

Each bonus should solve a SPECIFIC secondary problem. Three types:

### Type 1: Solve the NEXT Problem
> "Depois que você conseguir [resultado principal], o próximo desafio será [problema seguinte]. Por isso incluímos [bônus]."

**Exemplo:**
```
BÔNUS: Masterclass "Escale Sem Quebrar"
Depois que você faturar seus primeiros R$ 50k/mês, o desafio muda: como escalar sem perder qualidade?
Essa masterclass resolve exatamente isso. Valor: R$ 1.997
```

### Type 2: Speed Up the Result
> "Pra você chegar em [resultado] MAIS RÁPIDO, criamos [bônus]."

**Exemplo:**
```
BÔNUS: Templates Prontos "Copie e Cole"
Em vez de criar tudo do zero, use nossos 47 templates testados e validados.
Economize semanas de trabalho. Valor: R$ 2.497
```

### Type 3: Reduce Effort & Sacrifice
> "Sabemos que [obstáculo] pode travar você. Por isso [bônus] elimina essa barreira."

**Exemplo:**
```
BÔNUS: Suporte VIP por 6 Meses
Travou em alguma etapa? Nossa equipe resolve pra você em até 24h.
Sem ficar perdido. Sem ficar sozinho. Valor: R$ 3.000
```

### Stacking Rule:
> Each bonus should be worth MORE than the price of the main offer individually. This makes the stack feel absurdly valuable.

## 5. Price Anchoring (10x Value Rule)

The total perceived value should be AT LEAST 10x the price.

### Anchoring sequence for sales page:

```
1. [VALOR REAL]
   "Se você contratasse cada parte separadamente, investiria R$ [soma alta]."

2. [DESCONTO IMPLÍCITO]
   "Mas você não vai investir R$ [soma alta]."

3. [SEGUNDO ANCHOR]
   "Na verdade, nem R$ [valor intermediário]."

4. [PREÇO REAL]
   "Seu investimento total hoje é de apenas R$ [preço]."

5. [COMPARAÇÃO]
   "Menos de R$ [valor diário] por dia. Menos que [comparação cotidiana]."
```

### Daily breakdown template:
```
R$ [preço] ÷ 365 = R$ [valor/dia]
Menos que um cafezinho por dia pra [resultado transformador].
```

## 6. Scarcity Types

### 6.1 Real Scarcity (Escassez Real)
```
Limitado a [número] vagas porque [razão legítima].
[Motivo operacional: suporte individual, mentoria ao vivo, capacidade].
```

### 6.2 Urgency (Urgência)
```
Esse preço é válido até [data/hora].
Depois, o investimento volta para R$ [preço cheio].
[Razão: turma nova, reajuste, fim da promoção].
```

### 6.3 Bonus Scarcity (Escassez de Bônus)
```
Os bônus exclusivos só estão disponíveis para os [número] primeiros.
Depois disso, a oferta continua — mas sem [bônus mais valioso].
```

### Rule: ALWAYS provide a REASON for scarcity. Unjustified scarcity feels manipulative.

## 7. Dream Outcome Description Framework

Use vivid, specific language to describe the transformation:

### Template:

```
Imagine acordar [descrição do dia ideal].
[Ação específica que fazem] sem [dor/frustração anterior].
[Resultado mensurável] enquanto [benefício emocional].

Seus [relação: clientes/alunos/seguidores] [ação positiva].
Sua [métrica] está em [número aspiracional].
E o melhor: você conseguiu isso sem [sacrifício que temiam].
```

### Rules:
1. Use present tense (como se já tivesse acontecido)
2. Include sensory details (o que veem, ouvem, sentem)
3. Mix tangible results with emotional states
4. Reference what they WON'T have to do anymore

---

## 8. Trimming and Stacking Strategy

### Trimming: Remove friction from the offer

Identifique TUDO que o cliente precisa fazer/ter/saber para obter resultado e elimine:

| Friction Point | Solution |
|---------------|----------|
| "Preciso aprender X primeiro" | Inclua módulo introdutório |
| "Não tenho ferramenta Y" | Inclua acesso/template |
| "Não sei por onde começar" | Inclua roadmap/checklist |
| "E se eu travar?" | Inclua suporte/comunidade |
| "Leva muito tempo" | Inclua atalhos/done-for-you |

### Stacking: Add layers that increase perceived value

```
Camada 1: Produto principal (curso/mentoria/serviço)
Camada 2: Ferramentas/templates que aceleram
Camada 3: Comunidade/suporte que sustenta
Camada 4: Bônus que resolvem problemas adjacentes
Camada 5: Garantia que remove todo risco
```
