# Gary Bencivenga — Persuasion Frameworks for Sales Pages

> Reference file for sales page copy generation. Claude should apply these frameworks
> for scoring copy quality, writing fascinations/bullets, and structuring proof-driven pages.

---

## 1. Persuasion Equation

```
                Urgency × Relevance × Proof × Desire
Persuasion = ─────────────────────────────────────────────
                           Friction
```

### Scoring each element (1-10):

| Element | Score 1-3 (Weak) | Score 4-6 (OK) | Score 7-10 (Strong) |
|---------|-----------------|-----------------|---------------------|
| **Urgency** | Sem razão pra agir agora | Prazo genérico | Escassez real + consequência de não agir |
| **Relevance** | Mensagem genérica | Parcialmente relevante | Fala diretamente com a dor/desejo do leitor |
| **Proof** | Apenas afirmações | Alguns depoimentos | Demonstração + dados + depoimentos + lógica |
| **Desire** | Benefício vago | Benefício claro | Resultado vívido + transformação emocional |
| **Friction** (↓) | Preço alto + processo complexo + sem garantia | Alguma fricção | Preço justo + processo simples + garantia forte |

### What increases each element:

**Urgency (+):**
- Prazo real com consequência
- Vagas limitadas com razão operacional
- Bônus com data de expiração
- Aumento de preço programado

**Relevance (+):**
- Linguagem exata do público (words they use)
- Dor específica, não genérica
- Segmentação por situação ("Se você é [tipo] que [situação]...")
- Exemplos do universo deles

**Proof (+):**
- Demonstrações em vídeo/screenshots
- Depoimentos com nome, foto, resultado específico
- Dados e números verificáveis
- Logos de mídia/clientes reconhecidos
- Estudos de caso detalhados

**Desire (+):**
- Resultado final vívido (future pacing)
- Contraste antes/depois
- Detalhes sensoriais
- Resultado emocional além do funcional

**Friction (-):**
- Simplificar processo de compra
- Garantia forte e clara
- FAQ que elimina dúvidas
- Preço ancorado (valor >> preço)
- Depoimentos que quebram objeções

## 2. Fascinations (Bullet Writing Technique)

Fascinations are curiosity-driven bullets that make the reader NEED to know the answer. Each bullet should create an open loop that only the product can close.

### 10 Fascination Formulas in PT-BR:

**1. O Segredo**
```
"O segredo de [resultado] que [autoridade] usa — e que [número]% das pessoas ignoram completamente"
```
**Exemplo:** "O segredo de copywriters que faturam R$ 50k/mês — e que 97% dos iniciantes ignoram completamente"

**2. O Erro**
```
"O erro de R$ [valor] que você provavelmente está cometendo com [área] neste exato momento"
```
**Exemplo:** "O erro de R$ 15.000 que você provavelmente está cometendo com seus anúncios neste exato momento"

**3. O Método Contraintuitivo**
```
"Por que [ação que parece errada] na verdade é o caminho mais rápido para [resultado desejado]"
```
**Exemplo:** "Por que REDUZIR seu preço na verdade é o caminho mais rápido para AUMENTAR seu lucro"

**4. O Número Específico**
```
"[Número específico] maneiras de [resultado] — a #[número] vai te surpreender"
```
**Exemplo:** "14 maneiras de dobrar suas vendas sem gastar mais com anúncios — a #7 vai te surpreender"

**5. A Descoberta**
```
"A descoberta [origem/ano] que está mudando a forma como [tipo de pessoa] [ação] — e por que você deveria prestar atenção"
```
**Exemplo:** "A descoberta de Stanford (2024) que está mudando a forma como empreendedores precificam — e por que você deveria prestar atenção"

**6. O Aviso**
```
"AVISO: [ação comum] pode estar [consequência negativa] sem você perceber"
```
**Exemplo:** "AVISO: seus posts no Instagram podem estar AFASTANDO clientes sem você perceber"

**7. A Pergunta Que Incomoda**
```
"Você sabia que [fato surpreendente]? (A maioria dos [tipo de pessoa] não sabe — e paga caro por isso)"
```
**Exemplo:** "Você sabia que 73% dos leads que você gera NUNCA são seguidos corretamente? (A maioria dos vendedores não sabe — e paga caro por isso)"

**8. O Atalho**
```
"Como [resultado] em [tempo curto] sem [sacrifício que temem] — usando apenas [ferramenta/método simples]"
```
**Exemplo:** "Como criar 30 dias de conteúdo em 2 horas sem bloqueio criativo — usando apenas um checklist de 1 página"

**9. A Comparação**
```
"A diferença entre [pessoa que falha] e [pessoa que tem sucesso]? [Uma coisa simples]"
```
**Exemplo:** "A diferença entre quem vende R$ 3k/mês e quem vende R$ 30k/mês? Uma única mudança no processo de fechamento"

**10. O Mito Desmascarado**
```
"A mentira que te contaram sobre [tema]: '[crença popular]' — e o que REALMENTE funciona"
```
**Exemplo:** "A mentira que te contaram sobre tráfego pago: 'quanto mais você investe, mais vende' — e o que REALMENTE funciona"

## 3. Reason-Why Copy

Always give a REASON for everything. The reader's brain constantly asks "why?" — answer before they ask.

### Apply to:

| Element | Reason-Why |
|---------|-----------|
| **Preço baixo** | "Por que tão barato? Porque [razão legítima]." |
| **Preço alto** | "Por que esse investimento? Porque [valor entregue]." |
| **Garantia** | "Por que ofereço essa garantia? Porque [confiança no produto]." |
| **Escassez** | "Por que limitado? Porque [razão operacional]." |
| **Bônus** | "Por que estou dando isso grátis? Porque [estratégia/missão]." |
| **Oferta especial** | "Por que agora? Porque [evento/motivo]." |

### Template:
```
Você pode estar se perguntando: "Por que [aspecto da oferta]?"
A resposta é simples: [razão honesta e específica].
```

## 4. The Big Idea

Every sales page needs ONE unifying concept — a single, compelling idea that ties everything together.

### Characteristics of a strong Big Idea:
1. **Surpreendente** — O leitor não esperava
2. **Simples** — Cabe em uma frase
3. **Memorável** — Gruda na mente
4. **Verificável** — Pode ser provada
5. **Relevante** — Conecta diretamente ao desejo do leitor

### Testing the Big Idea:
> "Se eu pudesse dizer APENAS UMA COISA ao meu cliente ideal, o que seria?"

### Template:
```
A Grande Ideia: [conceito em 1 frase]
Reformulada como headline: "[headline que comunica a Big Idea]"
Prova principal: [evidência mais forte que sustenta a ideia]
```

## 5. Think Like a Fish

> "A isca deve agradar o PEIXE, não o pescador."

Write from the reader's perspective, not yours. Everything on the page should be about THEM.

### Checklist:
- [ ] A headline fala do LEITOR, não do produto?
- [ ] Os benefícios são sobre a VIDA DELE, não sobre features?
- [ ] A linguagem é a que ELE usa, não jargão técnico?
- [ ] As provas mostram pessoas COMO ELE, não celebridades distantes?
- [ ] O CTA fala do que ELE ganha, não do que você quer?

### Word swap rule:

| Replace (You-centric) | With (Reader-centric) |
|----------------------|----------------------|
| "Eu criei..." | "Você vai receber..." |
| "Meu método..." | "Seu resultado..." |
| "Nosso sistema..." | "Sua transformação..." |
| "Nós oferecemos..." | "Você terá acesso a..." |

## 6. Proof Hierarchy

Organize proof from STRONGEST to WEAKEST:

```
1. DEMONSTRAÇÃO (mais forte)
   "Veja funcionando ao vivo / screenshots / vídeo do resultado"

2. DEPOIMENTO ESPECÍFICO
   "Maria Silva, contadora, São Paulo: 'De R$ 3k para R$ 18k/mês em 4 meses'"

3. DADOS E NÚMEROS
   "92% dos nossos alunos atingem o resultado em 60 dias (pesquisa com 847 participantes)"

4. LÓGICA / RAZÃO
   "Se cada template economiza 2 horas, e são 47 templates, são 94 horas economizadas"

5. AFIRMAÇÃO (mais fraco)
   "Esse é o melhor curso do mercado"
```

### Rule: NEVER rely on claims alone. Always stack at least 3 levels of proof.

### Proof placement on page:
- After EVERY major claim, insert proof within 2 paragraphs
- Use the strongest available proof type for the most important claims
- Vary proof types to avoid repetition

## 7. Split-Test Thinking

Always provide testable alternatives. For every key element, generate at least 2 versions.

### Elements to always test:

| Element | Version A | Version B |
|---------|-----------|-----------|
| **Headline** | Benefit-driven | Curiosity-driven |
| **Lead** | Story opening | Data/stat opening |
| **CTA** | Action ("Comece agora") | Benefit ("Quero meu resultado") |
| **Guarantee** | Unconditional | Conditional (stronger) |
| **Price presentation** | Total upfront | Daily breakdown |
| **Social proof** | Numbers | Individual story |

### Application instruction:
> When generating sales page copy, ALWAYS create 2-3 headline variations and note which persuasion lever each one pulls (curiosity, benefit, proof, urgency).
