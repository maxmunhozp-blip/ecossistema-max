# FAB Framework (Features, Advantages, Benefits)

<!-- SYSTEM: FAB connects product attributes to emotional outcomes. Always progress F→A→B. Never list features without translating to benefits. -->

---

## FEATURE FORMULAS (5 Types)

### 1. Descriptive Feature
```
(Produto) inclui (característica específica) com (detalhe técnico).
```
```
Dentro do (produto), você encontra (quantidade) (módulos/aulas/templates/ferramentas) sobre (tema).
```

### 2. Quantified Feature
```
São (número) (módulos/aulas/horas/templates) cobrindo (tema) de forma (adjetivo).
```
```
(Produto) entrega (número) (recursos) + (número) (bônus) em (formato).
```

### 3. Comparative Feature
```
Diferente de (alternativa comum), (produto) oferece (característica exclusiva).
```
```
Enquanto (concorrentes/cursos genéricos) entregam (coisa básica), aqui você recebe (diferencial).
```

### 4. Composition Feature
```
(Produto) é composto por (componente 1) + (componente 2) + (componente 3), tudo integrado em (formato).
```
```
A estrutura completa inclui: (item 1), (item 2), (item 3) e (item 4) — tudo dentro de (plataforma/área de membros).
```

### 5. Process Feature
```
(Produto) funciona em (número) etapas: (etapa 1), (etapa 2), (etapa 3).
```
```
O método segue uma sequência lógica: primeiro (passo 1), depois (passo 2), e por fim (passo 3).
```

## ADVANTAGE FORMULAS (5 Types)

### 1. Comparative Advantage
```
Isso significa que, ao contrário de (alternativa), você (vantagem concreta).
```
```
Com isso, você não precisa mais (ação frustrante que a concorrência exige).
```

### 2. Elimination Advantage
```
Isso elimina completamente a necessidade de (problema/etapa desnecessária).
```
```
Você nunca mais vai precisar (ação dolorosa) porque (característica) já resolve isso.
```

### 3. Efficiency Advantage
```
O que antes levava (tempo antigo), agora leva (tempo novo) — (percentual)% mais rápido.
```
```
Em vez de gastar (tempo/dinheiro) com (método antigo), você (resultado) em (fração do tempo).
```

### 4. Exclusivity Advantage
```
Esse (recurso/método) é exclusivo do (produto) — você não encontra em nenhum outro (tipo de solução).
```
```
Isso foi desenvolvido especificamente para (público), baseado em (credencial/experiência).
```

### 5. Completeness Advantage
```
Tudo o que você precisa para (resultado) está reunido em um só lugar — sem precisar comprar (complemento).
```
```
É a solução completa: (componente 1) + (componente 2) + (componente 3), tudo junto e integrado.
```

## BENEFIT FORMULAS (5 Types)

### 1. Direct Benefit
```
O resultado prático é que você (resultado tangível e mensurável).
```
```
Na prática, isso faz você (ação) com (resultado) em (prazo).
```

### 2. Emotional Benefit
```
Imagine a sensação de (emoção positiva) quando você finalmente (conquista desejada).
```
```
Você vai sentir (emoção) ao perceber que (situação transformada).
```

### 3. Identity Benefit
```
Você deixa de ser (identidade atual negativa) e se torna (identidade aspiracional).
```
```
As pessoas vão te olhar diferente — como alguém que (identidade desejada pelo público).
```

### 4. Elimination Benefit
```
Nunca mais você vai (dor/frustração que será eliminada).
```
```
Acabou aquela sensação de (emoção negativa) toda vez que (situação recorrente).
```

### 5. Possibility Benefit
```
A partir de agora, você pode (possibilidade que antes parecia impossível).
```
```
Isso abre a porta para (possibilidade 1), (possibilidade 2) e até (possibilidade 3).
```

## INTEGRATION TEMPLATES

### FAB in Single Block
```
(Produto) inclui (feature específica).
Isso significa que você (advantage concreta).
E o resultado? Você (benefit emocional/prático).
```

### FAB in Running Sentence
```
Com (feature), você consegue (advantage), o que na prática significa (benefit).
```

### FAB in Bullet Points
```
✅ (Feature) → para que você (advantage) → e finalmente (benefit)
✅ (Feature) → sem precisar (eliminação) → conquistando (benefit)
✅ (Feature) → diferente de (alternativa) → garantindo (benefit)
```

### FAB for Sales Page Section
```
## (Título orientado ao benefit)

Dentro do (produto), você encontra (feature detalhada).

Isso é diferente de tudo que você já viu porque (advantage com comparação).

Na prática, isso significa que você vai (benefit direto) — sem (dor eliminada), sem (frustração eliminada), sem (obstáculo eliminado).

Imagine (cena visual do benefit emocional).
```

### FAB in Table
```
| O que você recebe | Por que isso importa | O que muda na sua vida |
|---|---|---|
| (Feature 1) | (Advantage 1) | (Benefit 1) |
| (Feature 2) | (Advantage 2) | (Benefit 2) |
| (Feature 3) | (Advantage 3) | (Benefit 3) |
```

## BANKS BY OFFER TYPE

### Courses (Cursos Online)
**Features:** Módulos gravados, aulas ao vivo, PDFs, templates, exercícios, quizzes, certificado, comunidade, suporte
**Advantages:** Assistir no seu ritmo, aplicar na hora, sem precisar de conhecimento prévio, acesso vitalício, atualizado constantemente
**Benefits:** Dominar (habilidade), gerar renda com (skill), confiança para (ação), liberdade de (situação)

### Mentoring (Mentoria)
**Features:** Calls ao vivo, acompanhamento individual, correções personalizadas, grupo exclusivo, hotseats, playbooks, acesso direto ao mentor
**Advantages:** Feedback personalizado, atalho de anos de tentativa e erro, rede de contatos qualificada, accountability
**Benefits:** Resultados acelerados, clareza total, segurança nas decisões, pertencer a um grupo de elite, transformação real (não só teoria)

### Digital Products (Produtos Digitais)
**Features:** Templates prontos, planilhas, scripts, checklists, frameworks, banco de prompts, automações
**Advantages:** Copiar e colar, não começar do zero, economizar tempo, testar rápido, plug-and-play
**Benefits:** Resultado imediato, mais tempo livre, profissionalismo instantâneo, eliminar procrastinação

### Events (Eventos/Imersões)
**Features:** Dias de imersão, networking, workshops práticos, painel de especialistas, materiais exclusivos, certificado
**Advantages:** Aprendizado intensivo, conexões presenciais, aplicação em tempo real, imersão total sem distrações
**Benefits:** Transformação em (X) dias, relacionamentos que mudam sua carreira, decisões que levam meses tomadas em horas

## WHEN TO USE TABLE FORMAT

<!-- SYSTEM: Use table format when there are 4+ features to present. Tables increase scannability and perceived value. Always use benefit-oriented column headers, never generic "Feature/Advantage/Benefit". -->

Use table format when:
- Listing 4+ features side by side
- Comparing your offer vs alternatives
- Building a "What's included" section
- Creating a quick-reference value summary
- The reader needs to scan and compare quickly

Avoid table format when:
- You have 1-3 FABs (use running blocks instead)
- The emotional narrative matters more than clarity
- You're in a storytelling section
- The benefit requires a visual scene (use paragraph)
